# RBG 推理服务部署最佳实践：概览与决策指南

## 本系列文档的定位

本系列共 5 篇文档，面向**推理服务工程师**，提供基于场景的操作指南。与 `doc/features/` 下的英文 Feature 文档相比：

| | Best Practice（本系列） | Feature 文档 |
|---|---|---|
| **目标** | 帮助用户做出正确的配置决策 | 完整的 API 参考和功能说明 |
| **组织方式** | 按部署场景串联 | 按功能模块独立 |
| **内容侧重** | 决策依据、权衡分析、常见问题 | 字段定义、参数说明、所有可选项 |
| **适合谁** | 第一次使用 RBG 部署推理服务的用户 | 需要查阅特定 API 字段的开发者 |

如需查阅完整的 API 参考，请参考 [Feature 文档目录](../TOC.md) 和 [API 参考](../reference/api.md)。

## 快速决策矩阵

根据你的部署场景，快速确定需要重点阅读的文档：

### 按模型规模和 GPU 数量

| 场景 | 角色架构 | 角色拓扑 | 必读文档 | 可选文档 |
|------|---------|---------|---------|---------|
| 小模型（< 10B），单 GPU | 聚合部署 | standalonePattern | 1 | 3, 4 |
| 中模型（10B-70B），2-4 GPU | 聚合部署 | leaderWorkerPattern | 1 | 2, 3, 4 |
| 大模型（> 70B），8+ GPU | 聚合部署 | leaderWorkerPattern | 1, 4, 5 | 2, 3 |
| 高吞吐场景，PD 分离 | PD 分离 | 按角色选择 | 1, 2, 3 | 4, 5 |
| 大模型 + PD 分离 | PD 分离 | leaderWorkerPattern | 1, 2, 3, 4, 5 | - |

### 按运维需求

| 运维需求 | 重点文档 |
|---------|---------|
| 首次部署，快速跑通 | 文档 1 |
| 减少 YAML 重复，规范化配置管理 | 文档 2 |
| 生产环境零停机升级 | 文档 3 + 文档 4 |
| PD 分离架构的版本一致性升级 | 文档 3（CoordinatedPolicy 章节） |
| 缩短大模型冷启动时间 | 文档 5 |
| 升级后 TTFT 指标波动优化 | 文档 4（原地升级 + KV Cache 章节） |
